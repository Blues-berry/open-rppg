# 可信生命信号相机 Demo

这个 Demo 用来展示“Self-Validating rPPG Camera”的产品概念：系统不直接输出心率，而是先完成采集可信校验。只有当人脸 ROI、光照稳定、运动抑制、颜色响应和 SQI 都达标时，才允许输出生理状态和安全事件。

## 运行方式

在仓库根目录启动本地静态服务：

```bash
python -m http.server 8010
```

然后打开：

```text
http://localhost:8010/demo/self-validating-camera/
```

浏览器允许摄像头权限后会使用真实画面；如果摄像头不可用，会自动进入模拟器模式。

## 演示路径

1. 点击“启动相机”。
2. 点击“运行可信校验”，观察设备自检、活体动作、多色响应、静止采样和证书生成。
3. 点击“模拟光照突变”或“模拟遮挡”，观察系统进入“不可判定”，并拒绝输出心率。
4. 点击“模拟风险趋势”，观察系统在可信采集下输出 L2/L3 安全事件。
5. 查看右下方 SDK JSON，理解它如何接入 DMS、安监平台或工位系统。

## 当前边界

这是产品交互 Demo，不是医疗设备，也不是完整 rPPG 算法实现。真实落地时，应把 `signals` 部分接入 Open-rppg 的实时推理结果，把当前前端中的质量门控和风险事件作为产品层逻辑。
